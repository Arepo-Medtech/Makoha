# Case-node authoring cheat-sheet (DERIVED — do not hand-edit)

> Generated by `scripts/schema-authoring-cheatsheet.mjs` from `data/schemas/*.schema.json`. The
> schema is authoritative; this is its exact shape made legible for authoring. Re-run after any
> schema change. Every `REQ` line MUST be present; every `enum`/`const` value MUST match exactly.
> `additionalProperties:false` is set on the nodes — a key not listed here is REJECTED.

Authoring lessons from the AUC-017 pilot (the traps that cost iterations):
- Many fields are OBJECTS or ARRAYS-OF-OBJECTS, not strings (e.g. `past_medical_history[]`,
  `patient_education_points[]`, `differential[]`, `diagnostic_reasoning_pitfalls[]`).
- `00.digital_tablet_anchoring.digital_tablet_version` is a **const** (see below) — NOT the omnibus
  version you used. `snomed_edition` is a const string, exact wording matters.
- Enums are strict: `certainty_achievable_via_telehealth`, `verificationStatus_fhir`, `position`,
  `edge_type`, `modifier_type`, `discipline`, `domain`, `care_plan_type`, `necessity`, `target_tier`.
- `scoring_rubric.minimum_domains_required` is an ARRAY of domain names, not a count.
